## 4. 当前最关键的结论

如果稳定版世界也没有出现：

```text
[探针] 稳定版API 2.8.0 脚本已加载
```

那么问题不是 NPC 项目、不是 UI、不是 `@minecraft/server-ui`、不是细手臂，也不一定是 Beta APIs。

`2.8.0` 是稳定版探针，不依赖 Beta API 入口。稳定版探针不显示意味着需要先定位“该客户端是否允许 Script 模块运行/是否把 world 内的 behavior_packs 作为有效脚本包加载”。

若稳定版探针显示、Beta 探针不显示，则问题才是 Beta API 权限或 beta API 版本接受性。

若两个探针都显示，才可继续将 NPC 项目的 API 依赖切换到与稳定版/Beta 探针相同的实际可运行版本。

## 5. 下一步测试，不改 NPC 项目

程序员或测试者应按此顺序仅记录结果：

1. 进入世界 `稳定版`，记录是否出现：

```text
[探针] 稳定版API 2.8.0 脚本已加载
```

2. 进入世界 `beta`，记录是否出现：

```text
[探针] Beta版API 2.9.0 脚本已加载
```

3. 分别截取两个世界的行为包“已激活”页面。
4. 在两个世界的游戏设置中确认“显示聊天”没有关闭或过滤系统消息。
5. 如果稳定版探针无提示，开启游戏 Creator 设置中的 Content Log 文件/GUI 输出（若该客户端提供此选项），重新进入世界，提交首个 script/module/manifest 报错。

在上述两个探针的结果明确前，禁止再修改 NPC 项目的 manifest、UI、皮肤或生成蛋名称。

## 6. 如何开始真正测试 NPC 项目

探针结果正常后，必须创建或选择一个新世界，并将 `customnpc` 的行为包 header UUID 写入该世界的 `world_behavior_packs.json`，资源包 header UUID 写入 `world_resource_packs.json`：

```text
BP header UUID: f127bcc0-501f-43c7-9690-60a3c4233666
RP header UUID: dc139b08-b19d-4c8a-894c-59be26ae7110
```

更安全的操作是在客户端世界编辑界面的“行为包”和“资源包”中手动启用“自定义NPC”与“自定义NPC资源包”，而非手写世界绑定 JSON。

绑定后再验证 NPC 启动探针：

```text
[自定义NPC] 脚本已加载 | API 2.9.0 / UI 2.2.0
```
