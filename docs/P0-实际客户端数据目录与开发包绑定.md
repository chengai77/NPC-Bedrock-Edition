# P0：实际 Minecraft 数据目录未加载自定义 NPC 包

> 本次不是 API、右键事件、细手臂或 UI 代码问题。
>
> 已直接检查 Windows 10 版 Minecraft UWP 的实际 LocalState 数据目录：
>
> ```text
> C:\Users\Chengai\AppData\Local\Packages\Microsoft.MinecraftUWP_8wekyb3d8bbwe\LocalState
> ```
>
> 该路径是链接，实际落点为 D 盘 WpSystem 数据目录。其 `games/com.mojang` 当前只有：
>
> ```text
> minecraftpe/
> resource_packs/DeadZzoneCn/
> ```
>
> **未发现**：
>
> ```text
> behavior_packs/
> development_behavior_packs/
> development_resource_packs/
> worlds/
> 自定义NPC BP/RP manifest
> world_behavior_packs.json
> world_resource_packs.json
> ```
>
> 结论：当前被检查的 Windows 10 版 Minecraft UWP 数据目录没有加载、也没有保存“自定义NPC”行为包。脚本探针不出现是必然结果。实体、物品或截图很可能来自另一套 Minecraft 客户端、另一个存储位置，或仅是未绑定行为包的数据残留。

## 1. 先确定实际运行的是哪个客户端

用户机器同时存在下列可能的 Minecraft 运行环境：

```text
C:\Users\Chengai\AppData\Local\Packages\Microsoft.MinecraftUWP_8wekyb3d8bbwe
C:\Users\Chengai\AppData\Roaming\Minecraft Bedrock
C:\Users\Chengai\AppData\Roaming\MinecraftPC_Netease_PB
C:\Users\Chengai\AppData\Roaming\MinecraftPE_Netease
```

程序员必须确认用户点击启动的客户端属于哪一套，不能只根据“Windows 10 版”推断。

验收方法：在目标客户端中打开“设置 -> 存储”，记录世界和资源包的实际数量；再在 Windows 中定位该客户端对应的 `games/com.mojang`。必须找到当前测试世界目录以及其 `world_behavior_packs.json` / `world_resource_packs.json`。

## 2. 使用开发包目录，禁止先依赖 mcaddon 导入缓存

在确认的实际客户端数据根目录下，创建或使用：

```text
games/com.mojang/development_behavior_packs/customnpc_BP/
games/com.mojang/development_resource_packs/customnpc_RP/
```

将项目目录直接复制为：

```text
D:\封装\草方块社区\自定义npc\BP
    -> <实际客户端>/games/com.mojang/development_behavior_packs/customnpc_BP

D:\封装\草方块社区\自定义npc\RP
    -> <实际客户端>/games/com.mojang/development_resource_packs/customnpc_RP
```

注意：目录结构必须是：

```text
customnpc_BP/manifest.json
customnpc_BP/scripts/main.js
customnpc_RP/manifest.json
```

不能多包一层 `BP/` 或 `RP/`，例如下面是错误结构：

```text
development_behavior_packs/customnpc_BP/BP/manifest.json
```

## 3. 新建世界并显式绑定两个开发包

1. 完全退出目标 Minecraft 客户端后再复制开发包。
2. 启动同一个客户端。
3. 新建世界，不使用旧世界。
4. 开启 **Beta APIs / 测试版 API**。
5. 在行为包页的“我的包”中找到 `自定义NPC`，点击启用，使其进入“已激活”。
6. 在资源包页的“我的包”中找到 `自定义NPC资源包`，点击启用，使其进入“已激活”。
7. 创建世界。

必须提交以下截图：

- 行为包“已激活”页：应显示 `自定义NPC` 和版本 `[1,1,1]` 或更高版本。
- 资源包“已激活”页：应显示 `自定义NPC资源包` 和相同版本。
- 实验页：Beta APIs 已开启。

## 4. 首个验证只看最小脚本探针

当前 `BP/scripts/main.js` 已是最小探针：

```js
import { world, system } from "@minecraft/server";

system.run(() => {
    world.sendMessage("[自定义NPC] 脚本已加载 | API 2.9.0 / UI 2.2.0");
});
```

进入刚创建的新世界后，聊天栏必须出现：

```text
[自定义NPC] 脚本已加载 | API 2.9.0 / UI 2.2.0
```

没有提示时，先确认世界绑定的开发包路径属于正在运行的那个客户端。不要重新打 `.mcaddon`，不要修改 UI、实体、渲染控制器或皮肤 registry。

## 5. 探针出现后的后续顺序

只有探针出现后：

1. 恢复 `skin_registry.js`。
2. 恢复 `npc_repository.js`。
3. 恢复 `npc_forms.js`。
4. 恢复 `npc_interaction.js`。
5. 最后恢复 `setupInteraction()`。

每次只恢复一个模块并进入同一新测试世界验证。任何一步使探针消失，即为该模块的实际阻断点。

## 6. 当前禁止事项

在确认实际客户端数据目录并显示启动探针前：

- 不再修改 API 版本。
- 不再修改 `skin_overrides.json`。
- 不再排查生成蛋名称语言键。
- 不再调试右键事件和表单。
- 不再反复导入同一个 `.mcaddon` 试图覆盖缓存。

当前 P0 是“附加包被放入/绑定到错误的 Minecraft 数据环境”，先让目标客户端真正加载 BP 和 RP。
