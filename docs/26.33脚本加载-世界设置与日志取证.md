# 26.33 脚本仍未加载：最终定位与测试门槛

> 前提事实已核实：当前 `自定义npc.mcaddon` 内实际包含 `BP/manifest.json`、`RP/manifest.json` 和最小 `BP/scripts/main.js`；manifest 内为 `min_engine_version [1,26,33]`、`@minecraft/server 2.9.0-beta.1.26.33-stable`、`@minecraft/server-ui 2.2.0-beta.1.26.33-stable`。不是打包旧文件。
>
> 实机结果：导入后仍未显示 `[自定义NPC] 脚本已加载`。
>
> 结论：此时不得继续修改 NPC、交互、皮肤、生成蛋名称或 UI 业务代码。脚本模块尚未被世界加载，必须先完成本文件的世界实验开关与 Content Log 取证。

## 1. 必须开启 Beta APIs

当前 1.26.33 使用的是 beta Script API：

```json
"@minecraft/server": "2.9.0-beta.1.26.33-stable"
"@minecraft/server-ui": "2.2.0-beta.1.26.33-stable"
```

这要求测试世界开启 **Beta APIs（测试版 API）** 实验开关。

操作要求：

1. 创建一个全新的测试世界。不要复用此前已经创建的世界。
2. 创建世界时进入“实验”设置。
3. 开启 **Beta APIs / 测试版 API**。
4. 同时在世界的行为包页启用“自定义NPC”，再到资源包页启用“自定义NPC资源包”。
5. 创建并进入该新世界。

如果不允许在已有世界中启用 Beta APIs，必须新建世界。仅导入 `.mcaddon` 不会自动给旧世界打开 beta Script API 权限。

## 2. 必须确认行为包和资源包都已绑定

实体能出现不代表脚本模块已经被世界启用。必须在世界编辑页确认：

- 行为包列表中的“已激活”区域有 `自定义NPC`，版本应为 `[1,1,1]` 或之后的递增版本。
- 资源包列表中的“已激活”区域有 `自定义NPC资源包`，版本必须与 BP 对 RP 的依赖版本一致。
- 不要只在“我的包”列表里看到它；必须处于“已激活”。
- 删除或禁用所有旧版本同 UUID 的“自定义NPC”行为包和资源包。

必须提供这两个页面的截图。没有截图，不接受“已经启用”的口头结论。

## 3. Content Log 是下一步唯一依据

如果新世界已开启 Beta APIs、BP/RP 均显示已激活，但启动探针仍不出现，下一步不要继续猜测。打开并复制 Content Log 中进入该世界时的**第一条**与以下关键词有关的错误：

```text
script
module
@minecraft/server
@minecraft/server-ui
manifest
dependency
main.js
customnpc
```

必须提交完整错误行及其前后各 20 行。不要只截取一句“failed”。

优先寻找下列错误类别：

1. beta API 未启用。
2. `@minecraft/server` 或 `@minecraft/server-ui` 不被当前游戏构建接受。
3. script module entry 无法加载。
4. manifest dependency / UUID / version 拒绝。
5. JavaScript 语法或模块导入错误。

## 4. 最小包对照实验

如果 Content Log 不可用，建立最小对照包来隔离“世界设置问题”与“当前附加包问题”。

对照包必须只保留：

```text
BP/manifest.json
BP/scripts/main.js
```

`main.js`：

```js
import { world, system } from "@minecraft/server";

system.run(() => {
    world.sendMessage("[Script Probe] loaded");
});
```

manifest 只保留 data module、script module 和 `@minecraft/server 2.9.0-beta.1.26.33-stable` 依赖。不要引用 RP、UI、实体、物品或其他 JS 文件。

测试结果解释：

- 对照包无提示：世界没有开启 Beta APIs，或 1.26.33 实际构建拒绝当前 beta API 版本。先解决环境/Content Log，不改 NPC 项目。
- 对照包有提示、NPC 包无提示：当前 NPC 包的 manifest、依赖或 JS 模块图仍有阻断；再逐项恢复最小 `main.js`，读取 Content Log 定位。

## 5. 明确当前禁止继续验证的项目

在 `[自定义NPC] 脚本已加载 | API 2.9.0 / UI 2.2.0` 出现前：

- 不验证右键 UI。
- 不验证皮肤槽位或细手臂。
- 不验证 AI、名称、命令、对话持久化。
- 不通过修改 `skin_overrides.json` 试图解决粗手臂。

这些功能全部依赖脚本成功加载；当前现象只是同一个根因的连带表现。
